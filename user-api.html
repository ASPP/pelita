User API Reference¶

This section summarizes the modules, classes and functions that are relevant when writing a player.

Convenience Properties of AbstractPlayer¶

Basic Properties

current_uni The current Universe.
current_state The current game state.
me The Bot object this Player controls.
team The Team object this Player’s Bot is on.
legal_moves The currently possible moves, and where they lead.

Positional Properties

initial_pos The initial_pos of this bot.
current_pos The current position of this bot.
previous_pos The previous position of the bot.

Team Properties

team_border Positions of the border positions.
team_bots A list of all Bots that are on this Player’s Bot’s Team.
other_team_bots A list of Bots that are on this Player’s team.
team_food Food owned by the team which can be eaten by the enemy Player’s bot.

Enemy Properties

enemy_team The enemy Team.
enemy_bots A list of enemy Bots.
enemy_food Food owned by the enemy which can be eaten by this Player’s bot.

Example Players¶

StoppingPlayer A Player that just stands still.
RandomPlayer A player that makes moves at random.
NQRandomPlayer Not-Quite-RandomPlayer that will move randomly but not stop or reverse.
BFSPlayer This player uses breadth first search to always go to the closest food.
BasicDefensePlayer A crude defensive player.

Accessing pelita.datamodel¶

Important Classes

CTFUniverse(maze, teams, bots) The Universe: representation of the game state.
Bot(index, initial_pos, team_index, homezone) A bot on a team.
Team(index, name, zone[, score, bots]) A team of bots.
Maze(width, height[, data]) A Mesh of strings of maze component representations.

Maze components

Wall str(object=’‘) -> string
Free str(object=’‘) -> string
Food str(object=’‘) -> string

Helper functions in pelita.graph¶

Convenience Functions

new_pos(position, move) Adds a position tuple and a move tuple.
diff_pos(initial, target) Return the move required to move from one position to another.
manhattan_dist(pos1, pos2) Manhattan distance between two points.

Interfacing with the CTFUniverse¶

Attributes

maze instance of pelita.datamodel.Maze
teams list of instances of pelita.datamodel.Team
bots list of instances of pelita.datamodel.Bot

Convenience Properties and Functions

bot_positions Current positions of all bots.
food_list Positions of all the food.
pretty Provide detailed, readable info about universe state.
enemy_team(team_index) Obtain the enemy team.
enemy_bots(team_index) Obtain enemy bot objects.
enemy_food(team_index) Food that is edible by a team
legal_moves(position) Obtain legal moves and where they lead.
other_team_bots(bot_index) Obtain other bots on team.
team_border(team_index) Positions of the border positions.
team_bots(team_index) Obtain all Bot objects in a Team.
team_food(team_index) Food that is owned by a team

Interfacing with the Maze¶

Convenience Properties and Functions

shape The shape (width, height) of the Mesh.
positions The indices of positions in the Maze.
pos_of(char) The indices of positions which have a maze component.
