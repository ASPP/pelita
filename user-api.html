User API Reference¶

This section summarizes the modules, classes and functions that are relevant when writing a player.

Convenience Properties of AbstractPlayer¶

Basic Properties

current_uni The current Universe.
current_state The current game state.
me The Bot object this Player controls.
team The Team object this Player’s Bot is on.
legal_moves The currently possible moves, and where they lead.

Positional Properties

initial_pos The initial_pos of this bot.
current_pos The current position of this bot.
previous_pos The previous position of the bot.

Team Properties

team_border Positions of the border positions.
team_bots A list of all Bots that are on this Player’s Bot’s Team including the current bot.
other_team_bots A list of Bots that are on this Player’s team excluding the current bot.
team_food Food owned by the team which can be eaten by the enemy Player’s bot.

Enemy Properties

enemy_team The enemy Team.
enemy_bots A list of enemy Bots.
enemy_food Food owned by the enemy which can be eaten by this Player’s bot.

Example Players¶

StoppingPlayer A Player that just stands still.
