class pelita.player.AbstractTeam[source]¶

Bases: object

Abstract team class.

Players who want to write their own Team class (instead of using SimpleTeam together with a number of independent AbstractPlayer subclasses), should inherit from AbstractTeam (though this is not strictly necessary).

class pelita.player.SimpleTeam(*args)[source]¶

Bases: pelita.player.AbstractTeam

Simple class used to register an arbitrary number of (Abstract-)Players.

Each Player is used to control a Bot in the Universe.

SimpleTeam transforms the set_initial and get_move messages from the GameMaster into _set_index, _set_initial and _get_move messages on the Player.

Parameters:
  • team_name – the name of the team (optional)
  • players – the Players who shall join this SimpleTeam

class pelita.player.TestPlayer(moves)[source]¶

Bases: pelita.player.AbstractPlayer

A Player with predetermined set of moves.

Parameters:moves (list of moves or str of shorthand symbols) – the moves to make in order, see notes below

Notes

The moves argument can either be a list of moves, e.g. [west, east, south, north, stop] or a string of shorthand symbols, where the equivalent of the previous example is: '><v^-'.
